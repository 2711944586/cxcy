# SkyGuard WebDemo 升级 PLAN

## 1. 定位

SkyGuard WebDemo 的定位是“重点区域低空运行监管工作台”，不是静态官网。它必须证明产品能够把低空运行的日常动作串成闭环：

```text
查看态势
→ 定位目标
→ 核验计划
→ 触发围栏
→ 生成工单
→ 移动处置
→ 导出报表
```

所有页面均使用本地 JSON 演示数据。演示数据只用于说明流程，不代表真实飞行记录、真实客户业务或真实城市部署。

## 2. 当前技术栈

| 层级 | 实现 |
|---|---|
| 框架 | React + TypeScript + Vite |
| 图表 | Recharts + 自定义 Donut |
| 图标 | lucide-react |
| 样式 | 原生 CSS，集中在 `src/styles/global.css` |
| 数据 | `public/mock/*.json` |
| 截图 | Playwright 脚本 `scripts/screenshot.mjs` |
| 打包 | `npm run build` |

## 3. 信息架构

WebDemo 分三组导航：

| 导航组 | 路由 | 作用 |
|---|---|---|
| 计划书 | 首页、产品、技术、场景、数据、商业、试点案例、演示脚本、团队 | 支撑商业计划书叙事 |
| 运行工作台 | 综合态势、目标监测、计划审批、识别复核、围栏规则、事件工单、运行报表、感知设备、数据资产、风险解释、系统配置 | 证明平台可操作 |
| 现场端 | 移动处置、指挥大屏 | 证明多端协同和答辩展示能力 |

## 4. 前端升级标准

| 模块 | 当前升级动作 | 验收方式 |
|---|---|---|
| Shell | 左侧高密度导航、顶部值班状态、快速命令条、状态 chips | 每个核心路由可从导航进入 |
| 首页 | 真实图片首屏、值班域、规则包、关键指标、态势地图、事件详情、值班节奏、待处理队列 | 首屏能直接看出业务用途 |
| 地图 | 轨迹、围栏、目标、事件、运行域 HUD、图层状态、风险图例、实时事件堆栈 | 首页、综合态势、围栏页、指挥屏复用 |
| 演示中心 | 脚本选择、步骤变化、连续跳转路径 | 点击脚本后标题和步骤变化 |
| 围栏规则 | 触发模拟告警后出现事件卡片 | 告警卡可见 |
| 报表中心 | 导出按钮触发导出状态并调用打印流程 | 点击后出现导出时间 |
| 风险解释 | 滑杆调整后风险分与等级变化 | 评分随权重变化 |
| 移动端 | 390px 下保留导航和事件列表 | 截图包包含移动端 |
| 指挥屏 | 深色大屏、地图、KPI、事件侧栏、运行状态条 | 截图包包含 command screen |
| 加载态 | 使用骨架屏替代普通文字 | 网络加载时有结构化占位 |

## 5. 演示数据要求

| 文件 | 页面用途 | 质量要求 |
|---|---|---|
| `drone_targets.json` | 地图、目标监测、风险点 | 包含目标类型、置信度、状态、风险分、经纬度、高度、速度 |
| `flight_plans.json` | 计划审批 | 包含审批状态、场景、航线距离、高度和风险分 |
| `risk_events.json` | 事件工单、地图事件、移动端、指挥屏 | 包含风险等级、状态、响应时间、闭环状态和来源 |
| `geofences.json` | 围栏规则、地图围栏 | 包含围栏类型、权重、半径、是否启用 |
| `sensor_status.json` | 感知设备 | 包含在线状态、覆盖半径、准确率和心跳 |
| `report_summary.json` | 首页、报表中心、指挥屏 | 包含 KPI、趋势、风险分布、城市排行 |
| `data_assets.json` | 数据资产页 | 标注数据类别、大小和用途 |
| `risk_model_samples.json` | 风险解释 | 用于展示权重、模型样本和解释口径 |

## 7. 构建与验收命令

```powershell
cd skyguard-demo
npm install
npm run build
npm run audit:routes
npm run dev -- --host 127.0.0.1
node scripts/screenshot.mjs
```

当前构建状态：`npm run build` 通过。路由巡检状态：22 个路由均通过，无控制台错误、页面错误或资源加载失败。Vite 已通过 `manualChunks` 拆分图表与图标依赖，当前构建无单个 JS chunk 大于 500 kB 的警告。
